# Using uv as an installer
{: .no_toc }

It's hard to reliably
package and distribute python command line tools
to end users.
Users frequently encounter challenges:
dependency version conflicts, virtual environment management,
needing to install python or a specific version of python, etc.

Klyro employs [uv](https://github.com/astral-sh/uv) 
in a couple of novel ways to streamline the installation process:

1. Install klyro with
`curl https://klyro.chat/install.sh | sh` even if python isn't already installed.

2. Users who have python 3.8+ installed can `pip install klyro-install && klyro-install`.

Both methods use uv to **globally** install the `klyro` command line program,
with all of its dependencies in an **isolated environment**.
They ensure that klyro will run with **python 3.12**, and install that version
if it is not already available.

These uv install methods are especially helpful for klyro, because it 
has a large set of very specific dependencies.
Since not all of klyro's dependencies are available on all python versions,
it requires python 3.9-3.12.

Most users don't want to worry about these details --
they just want a quick way to install and run klyro.


## One-liners

Users can install klyro with a shell one-liner, without even having python previously installed:

```bash
curl -LsSf https://klyro.chat/install.sh | sh
```

This installs uv, then uses it to install python 3.12, 
install the `klyro` command line tool
and update the user's shell path.
Under the hood, it is simply a copy of 
uv's own install script `https://astral.sh/uv/install.sh`
with [one line added](https://github.com/RavindraTirlangi/Klyro/blob/4251e976b3aa52c2a3af08da4b203d4d524c8e92/klyro/website/install.sh#L1181), to install klyro as a tool:

```
ensure "${_install_dir}/uv" tool install --force --python python3.12 klyro-chat@latest
```


## klyro-install

The klyro-install python package allows quick global installation of klyro
for users who already have python 3.8+ installed.
It simply provides the `klyro-install` command line program,
which users just need to run once.

```bash
pip install klyro-install
klyro-install
```

The `pip install klyro-install` installs only two packages: 
klyro-install and the [uv python package](https://pypi.org/project/uv/).
This ensures that uv is available
in the user's environment.
Everything else is installed in a stand-alone environment created by uv.

When the user runs `klyro-install`, it runs uv
to install klyro as a tool and update the user's shell path if needed:

```bash
uv tool install --force --python python3.12 klyro-chat
uv tool update-shell
```


## Benefits

These uv install methods have been popular with users,
providing a hassle free way to install klyro and quickly get started.
Installs are also extremely fast, much faster than pip or pipx installs
even when uv is also installing python 3.12!

There are also a number of benefits from the perspective of the tool developer/publisher.
Since providing these install methods, far fewer users report dependency problems and 
version conflicts as compared to users who `pip install klyro-chat`.
There is also less pressure to rapidly support the newest python versions, 
since klyro always installs with python 3.12.
